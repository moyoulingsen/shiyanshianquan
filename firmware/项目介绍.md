# LabGuard TwinP4 项目介绍

## 一、项目概述

`LabGuard TwinP4` 是一套面向高校和科研实验室场景的智能安全系统。系统采用两块 `ESP32-P4-Function-EV-Board` 分工协同，形成“门外准入审核 + 门内危险监测 + 联动处置 + 网络告警”的完整闭环。

- 门外节点负责人员靠近唤醒、PPE 穿戴检测、门禁判定、语音提示、灯光反馈、触摸屏显示。
- 门内节点负责烟雾/火焰视觉检测、温湿度/VOC/烟雾传感采集、风险融合判断、风扇/水泵等执行器联动。
- 双节点通过 Wi‑Fi 与 MQTT 协同工作，实现门内风险状态对门外准入逻辑的实时约束。

项目目标不是做一个单点报警器，而是构建一套覆盖“进入前预防、实验中监测、异常后联动”的实验室安全闭环系统。

## 二、应用场景与痛点

实验室管理常见问题主要集中在三类：

1. 人员进入实验室前未按要求穿实验服、护目镜等 PPE。
2. 实验进行中出现烟雾、火焰、温度异常、空气质量恶化等风险，传统人工巡查响应慢。
3. 门禁、监测、告警、联动处置通常彼此割裂，缺少统一状态联动。

本项目针对这些问题，将门禁准入和危险监测结合到一个统一系统中，做到：

- 门内危险时自动禁止继续入内。
- 门外 PPE 不合格时禁止开门并即时提示。
- 室内出现异常时自动触发风扇、水泵、声光告警。
- 关键状态和事件可通过 MQTT 上报，便于网页端或上位机接入。

## 三、系统架构

### 1. 门外准入节点

门外节点部署在实验室入口，主要硬件包含：

- `ESP32-P4-Function-EV-Board`
- `OV2740 MIPI CSI` 摄像头
- `7 英寸 MIPI DSI 电容触摸屏`
- `RCWL-0516` 靠近唤醒模块
- `SG90` 舵机模拟门锁
- `WS2812B` 灯环
- 小喇叭

主要功能：

- 检测人员是否靠近。
- 判断实验服、护目镜是否佩戴齐全。
- 接收门内风险状态。
- 给出“允许进入 / 拒绝进入 / 危险禁入”的门禁判定。
- 通过屏幕、语音和灯效进行交互提示。

### 2. 门内风险监测节点

门内节点部署在实验室内部，主要硬件包含：

- `ESP32-P4-Function-EV-Board`
- `OV2740 MIPI CSI` 摄像头
- `SHT30` 温湿度模块
- `ENS160` VOC/空气质量模块
- `MQ-2` 烟雾/可燃气体模块
- `5V` 风扇
- `5V` 微型水泵
- `4 路 MOSFET` 驱动板

主要功能：

- 采集室内温湿度、VOC、烟雾告警输入。
- 对室内图像进行烟雾/火焰识别。
- 融合多源信息生成风险等级。
- 根据风险等级执行风扇、水泵、报警联动。
- 把风险状态同步给门外节点。

### 3. 通信与协同

两块节点接入同一 `2.4GHz Wi‑Fi` 网络，通过 MQTT 交换状态：

- 门内节点发布 `sensor` 和 `risk`。
- 门外节点订阅门内风险，发布 `ppe` 和 `access`。
- 双节点都发布 `status` 和 `event`。
- 管理端可通过 `cmd/reset`、`cmd/test` 等 topic 下发控制命令。

## 四、软件设计

## 1. 软件目录

```text
firmware/
├─ components/
│  ├─ labguard_common/
│  ├─ labguard_net/
│  └─ event_log/
├─ esp_outdoor/
│  ├─ main/
│  └─ components/
└─ esp_indoor/
   ├─ main/
   └─ components/
```

## 2. 门外节点软件模块

- `camera_capture/`：门外摄像头抽象层。
- `ppe_infer/`：PPE 检测模块，当前实现为可切换场景的演示模拟器，后续可替换为 ESP-DL 模型。
- `door_fsm/`：门禁状态机，根据 PPE 和室内风险生成准入决策。
- `ui_lvgl/`：触摸屏信息展示接口。
- `audio_prompt/`：语音提示接口。
- `led_status/`：灯效状态接口。

## 3. 门内节点软件模块

- `camera_capture/`：门内摄像头抽象层。
- `sensor_reader/`：室内传感器采集接口，当前实现为可切换场景模拟。
- `hazard_infer/`：烟雾/火焰识别接口，当前实现为可切换场景模拟。
- `risk_fusion/`：风险融合模块，综合视觉与传感器结果。
- `actuator_ctrl/`：风扇、水泵、报警等联动接口。

## 4. 公共组件

- `labguard_common/`
  - 定义状态结构体。
  - 提供 JSON 编码/解码。
  - 定义 MQTT topic、命令类型、风险等级。
- `labguard_net/`
  - 封装 Wi‑Fi 与 MQTT。
  - 网络配置未设置时自动退化为本地日志模式，便于无网演示。
- `event_log/`
  - 记录最近事件。
  - 统一事件发布格式，便于后续接入 TF 卡或网页端日志。

## 五、核心流程

### 1. 门外准入流程

1. 摄像头采集人员画面。
2. `ppe_infer` 输出是否有人、是否穿实验服、是否戴护目镜。
3. 同时读取门内节点最新风险等级。
4. `door_fsm` 根据 PPE 结果与室内风险生成准入结论。
5. 若允许进入，则开锁并播报通过提示。
6. 若不允许，则保持锁定并给出拒绝原因。

### 2. 门内风险处置流程

1. 读取温湿度、VOC、MQ-2 烟雾告警状态。
2. 读取门内摄像头识别结果。
3. `risk_fusion` 综合判断：
   - `normal`
   - `warning`
   - `alarm`
   - `emergency`
4. `actuator_ctrl` 按等级执行联动：
   - `warning`：仅提示
   - `alarm`：报警 + 风扇
   - `emergency`：报警 + 风扇 + 水泵
5. 发布 `risk` 状态给门外节点和上位机。

### 3. 命令控制

系统支持通过 MQTT 切换演示或测试场景：

- `reset`
- `selftest`
- `force_normal`
- `force_warning`
- `force_alarm`
- `force_emergency`
- `admin_lock`
- `admin_unlock`

这使得比赛演示时不需要真的点火或制造烟雾，也能完整展示业务链路。

## 六、创新点

### 1. 双节点协同闭环

系统不是单独做“门口识别”或“室内报警”，而是把实验室安全分成门外与门内两个职责明确的节点，实现完整联动闭环。

### 2. 多源信息融合

风险判断同时考虑视觉识别、温度、VOC、MQ-2 告警等多源信息，比单一阈值触发更稳健。

### 3. 强交互性

系统包含触摸屏界面、语音播报、灯光提示、门禁动作和网络消息，具备完整的 AIoT 人机交互链路。

### 4. 演示与实装兼容

当前代码已支持“模拟场景演示模式”，同时保留了摄像头、传感器、执行器、LVGL、音频的清晰接口，便于比赛前快速演示，也便于后续接真实硬件。

## 七、硬件清单对应关系

采购清单中的关键器件与系统映射如下：

- `ESP32-P4-Function-EV-Board ×2`：门外节点、门内节点主控
- `OV2740 ×2`：门外 PPE 检测、门内烟雾火焰检测
- `7 寸触摸屏 ×1`：门外交互界面
- `RCWL-0516 ×1`：门外靠近唤醒
- `SHT30 ×1`：温湿度采集
- `ENS160 ×1`：空气质量 / VOC
- `MQ-2 ×1`：烟雾 / 可燃气体辅助检测
- `SG90 ×1`：门锁演示执行器
- `5V 风扇 ×1`：通风执行器
- `5V 水泵 ×1`：喷淋演示执行器
- `WS2812B ×1`：状态灯效
- `MOSFET 驱动板 ×1`：风扇/水泵/灯带驱动

## 八、后续落地建议

当前仓库已经具备完整业务逻辑，但真实硬件驱动仍建议按下面顺序替换：

1. 先把 Wi‑Fi/MQTT 参数改为 `menuconfig` 配置。
2. 对接真实 `OV2740` 采集驱动。
3. 用真实 `SHT30 / ENS160 / MQ-2` 替换传感器模拟数据。
4. 用真实 `GPIO/PWM/MOSFET` 实现舵机、风扇、水泵、WS2812B 控制。
5. 把 `ppe_infer` 和 `hazard_infer` 替换成 ESP-DL 模型推理。
6. 把 `event_log` 扩展为 TF 卡存储。

## 九、总结

`LabGuard TwinP4` 通过双 `ESP32-P4` 协作，把实验室门禁准入、室内危险识别和自动联动处置整合为一个完整 AIoT 系统。项目既适合课程设计、比赛演示，也适合作为后续真实实验室安全终端的原型基础。
